    1. Create `apps/api/src/auth/auth.service.ts`:
       - Inject SUPABASE_ADMIN_TOKEN and DB_TOKEN
       - `createAnonymousUser()` method:
         a. Call `supabase.auth.admin.createUser({ user_metadata: { is_anonymous: true } })` — creates user with no email/password
         b. The admin createUser in Supabase JS v2 does NOT return a session. To get a JWT, use a workaround: call `supabase.auth.admin.generateLink({ type: 'magiclink', email: '<userId>@anon.wonderwaltz.local' })` and extract the token from the hashed_token property, OR better: use `supabase.auth.signInWithPassword` is not available for anon users. The cleanest approach: after creating the user, use Supabase's `auth.admin.createUser` with `email_confirm: true` and then generate a session via `auth.admin.generateLink`. If this approach does not yield a usable JWT, fall back to generating a custom signed JWT using the Supabase JWT secret (from `SUPABASE_JWT_SECRET` env var) with the `jsonwebtoken` library (already available via NestJS deps) — payload: `{ sub: userId, aud: 'authenticated', role: 'authenticated', iss: SUPABASE_URL + '/auth/v1', iat, exp: iat + 3600 }`.
