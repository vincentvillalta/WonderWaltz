From RESEARCH.md — v1 API surface to design:

INGESTION READS (live data — real implementations Wave 3):
```
GET /v1/parks — list parks
GET /v1/parks/:parkId/waits — { attractionId, name, minutes, fetched_at, source, is_stale }[]
GET /v1/crowd-index — { global: CrowdIndex, parks: Record<ParkId, CrowdIndex> }
  CrowdIndex = { value: number | null, confidence: 'bootstrap' | 'percentile', sample_size_days: number }
GET /v1/weather?date=YYYY-MM-DD — { high_f, low_f, condition, precipitation_pct, humidity_pct, uv_index } | null
```

PHASE 3 STUBS (501 Not Implemented):
```
POST /v1/trips — body: { resort_id?, start_date, end_date, guests: GuestInput[], preferences: TripPreferences }
GET /v1/trips/:id — Trip object
POST /v1/trips/:id/generate-plan — body: {} → { job_id: string }
GET /v1/plans/:id — Plan object with day_plans: DayPlan[]
POST /v1/trips/:id/rethink-today — body: { remaining_items: string[] } → { job_id: string }
```

PHASE 4 STUBS (501 Not Implemented):
```
POST /v1/auth/anonymous — body: {} → { access_token: string, user_id: string, expires_at: string }
GET /v1/users/me — User object
```

Core shape requirements (from REQUIREMENTS.md SOLV-03, PLAN-02):
- GuestInput: { name: string, age_bracket: '0-2'|'3-6'|'7-9'|'10-13'|'14-17'|'18+', has_das: boolean }
- TripPreferences: { budget_tier: 'pixie_dust'|'fairy_tale'|'royal_treatment', must_do_attraction_ids: string[], mobility_notes?: string }
- Trip: { id, resort_id?, start_date, end_date, guests, preferences, entitlement_state: 'free'|'unlocked', created_at }
- DayPlan: { id, date, park_id, items: PlanItem[], crowd_index?, weather? }
- PlanItem: { id, type: 'attraction'|'dining'|'show'|'break'|'travel', ref_id?, name, start_time, end_time, wait_minutes?, notes?, lightning_lane_type?: 'LLMP'|'LLSP'|null }
- Plan: { id, trip_id, version, status: 'generating'|'ready'|'failed', day_plans: DayPlan[], created_at }
