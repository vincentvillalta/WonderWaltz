  <behavior>
    - `generate(tripId): Promise<{ planId, cached: boolean }>`
      1. Load trip + guests + preferences from DB.
      2. Compute `solverInputHash` (BEFORE hydrating forecasts — forecasts excluded from hash).
      3. Query `plans WHERE trip_id = $1 AND solver_input_hash = $2 LIMIT 1`.
         - If hit: update `trips.current_plan_id`, return `{ planId, cached: true }`. Zero LLM cost.
      4. Hydrate: catalog (attractions/dining/shows from DB cache OR YAML), walking graph (from loader), forecasts for every candidate ride/slot, weather for each date, crowd buckets.
      5. Call `solve(solverInput)`.
      6. Call `NarrativeService.generate(...)`.
      7. Call `PersistPlanService.persist(...)` → returns planId.
      8. Update `trips.current_plan_id` and `trips.plan_status = 'ready'`.
      9. Return `{ planId, cached: false }`.
    - On BudgetExhaustedError: update `trips.plan_status = 'failed'` and re-throw (BullMQ retry will NOT retry on this — use `UnrecoverableError`).
    - Timing: each step timestamped; total < 30s for 5-day fixture (snapshot fixture from 03-10).
  </behavior>
  <action>

       - Happy path: 3-day fixture → returns `{ planId, cached: false }`; ≤ 5s runtime (test harness).
       - Cache hit: pre-seed DB with matching hash → returns `{ cached: true }`; zero Anthropic mock calls.
       - BudgetExhaustedError → throws `UnrecoverableError`; `trips.plan_status = 'failed'`.
