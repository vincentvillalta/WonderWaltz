  <action>
    Create SwiftData `@Model` classes for the offline data package. These mirror the API response shapes but are persisted locally.

    **CRITICAL SwiftData rules for Swift 6 strict concurrency:**
    - `@Model` objects are NOT Sendable. Never pass them across actor boundaries — pass `PersistentIdentifier` instead.
    - Use `#Predicate` for queries, not closures.
    - Avoid `.count` on array relationships (iOS 18 memory issue). Use separate `FetchDescriptor` with `.fetchCount`.
    - Use `@Attribute(.externalStorage)` only for large blobs (map tiles).

    1. `CachedTrip.swift` — `@Model` with:
       - `id: String` (server trip ID), `startDate: Date`, `endDate: Date`, `status: String`
       - `parkIds: [String]`, `hasHopper: Bool`, `budgetTier: String`
       - `entitlementState: String` (locked/unlocked)
       - `lastSyncedAt: Date`
       - Relationship: `@Relationship(deleteRule: .cascade) var plans: [CachedPlan]`

    2. `CachedPlan.swift` — `@Model` with:
       - `id: String` (server plan ID), `tripId: String`, `status: String`, `version: Int`
       - `forecastDisclaimer: String?` (Beta Forecast framing text — FC-05)
       - `lastSyncedAt: Date`
       - `@Relationship(deleteRule: .cascade) var days: [CachedPlanDay]`
       - Inverse relationship to CachedTrip

    3. `CachedPlanDay.swift` — `@Model` with:
       - `id: String`, `dayNumber: Int`, `date: Date`, `parkId: String`, `parkName: String`
       - `narrativeIntro: String?`, `narrativeTip: String?`
       - `isLocked: Bool` (true for Days 2+ on free tier)
       - `@Relationship(deleteRule: .cascade) var items: [CachedPlanItem]`

    4. `CachedPlanItem.swift` — `@Model` with:
       - `id: String`, `type: String` (attraction/meal/show/ll_reminder/rest/walk)
       - `refId: String?` (catalog ID), `name: String`, `startTime: String`, `endTime: String`
       - `waitMinutes: Int?`, `walkMinutes: Int?`, `narrativeTip: String?`
       - `sortOrder: Int`
       - Relationship to CachedPlanDay (inverse)

    5. `CachedAttraction.swift` — `@Model` for catalog subset:
       - `id: String` (attraction UUID), `name: String`, `parkId: String`
       - `latitude: Double?`, `longitude: Double?`
       - `heightRequirementInches: Int?`, `tags: [String]`
       - `lightningLaneType: String?`

    6. `CachedWalkingEdge.swift` — `@Model` for walking graph:
       - `fromId: String`, `toId: String`, `walkMinutes: Double`
       - Use compound uniqueness on (fromId, toId)

    7. `WizardDraft.swift` — `@Model` for auto-save per CONTEXT.md:
       - `startDate: Date?`, `endDate: Date?`
       - `selectedParkIds: [String]`, `hasHopper: Bool`
       - `guestsJSON: String` (serialize guest array as JSON string — avoids nested @Model complexity)
       - `budgetTier: String?`, `lodgingType: String?`, `transportType: String?`
       - `mustDoRideIds: [String]`, `mealPreferences: [String]`
       - `currentStep: Int` (0-7, resume index)
       - `createdAt: Date`, `updatedAt: Date`

    8. `ModelContainerConfig.swift` — factory for the shared ModelContainer:
       ```swift
       public enum ModelContainerConfig {
           public static func makeContainer() throws -> ModelContainer {
               let schema = Schema([
                   CachedTrip.self, CachedPlan.self, CachedPlanDay.self,
                   CachedPlanItem.self, CachedAttraction.self,
                   CachedWalkingEdge.self, WizardDraft.self
               ])
               let config = ModelConfiguration(isStoredInMemoryOnly: false)
               return try ModelContainer(for: schema, configurations: [config])
           }
       }
       ```

    1. Create `OfflineStore.swift` — concrete implementation of `OfflineStoreProtocol`:
       - Uses `@ModelActor` for background SwiftData operations (never blocks main thread)
       - `cachePlan(planResponse:)`: maps API response to CachedPlan + CachedPlanDay + CachedPlanItem models, inserts in background context
       - `getCachedPlan(id:)`: fetches by plan ID, returns lightweight struct (not @Model object) for cross-actor safety
       - `isCached(planId:)`: lightweight existence check via FetchDescriptor with fetchLimit: 1
       - `clearCache()`: deletes all cached data
       - `saveWizardDraft(_:)`: upserts WizardDraft (delete old + insert new)
       - `loadWizardDraft() -> WizardDraftData?`: loads the most recent draft as a plain struct
       - `cacheAttractions([AttractionData])`: bulk insert CachedAttraction models
       - `cacheWalkingGraph([WalkingEdgeData])`: bulk insert CachedWalkingEdge models

       Define plain data structs (CachedPlanData, WizardDraftData, AttractionData, WalkingEdgeData) for cross-actor transfer. These are Sendable value types.

    2. Create `SyncCoordinator.swift` — `@Observable` class managing sync state:
       - `var isSyncing: Bool`
       - `var lastSyncDate: Date?`
       - `var isOffline: Bool` (monitored via NWPathMonitor)
       - `func syncIfNeeded(tripId: String)`: checks if cached data is stale (> 1 hour old), fetches fresh data from API if online, updates SwiftData cache
       - `func startMonitoringNetwork()`: uses NWPathMonitor to track connectivity changes. When connection resumes after being offline, triggers sync
       - Per CONTEXT.md: when significant changes detected (park hours updated), sets `pendingRefreshPrompt = true` with message "Park hours updated -- refresh your plan?" — never auto-refreshes without user confirmation
       - Inject `APIClientProtocol` and `OfflineStore` via init (protocol-based DI)

    3. Create `OfflinePackageDownloader.swift`:
       - `func downloadOfflinePackage(for planId: String)`: triggered automatically after plan generation
       - Downloads: catalog subset (attractions referenced in plan), walking graph edges between plan locations, static map tiles for park areas
       - Uses background URLSession for downloads (continues if app is backgrounded)
       - Stores structured data via OfflineStore (SwiftData), map tiles via FileManager in app's caches directory
       - Sets `isDownloadComplete` flag on cached plan when done
       - Shows small "Downloaded for offline" badge (via a published property the plan view can observe)
       - Inject `APIClientProtocol` for data fetching, `OfflineStore` for persistence
