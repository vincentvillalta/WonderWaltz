    **WWCore — Networking:**

    1. Copy `packages/shared-openapi/openapi.v1.snapshot.json` to `apps/ios/Packages/WWCore/Sources/WWCore/openapi.json`. This is the spec Swift OpenAPI Generator will consume at build time.

    2. Create `openapi-generator-config.yaml` next to it:
       ```yaml
       generate:
         - types
         - client
       accessModifier: public
       ```
       Both files must be in the target's source directory so the build plugin finds them.

    3. Create `APIClient.swift` — concrete implementation of `APIClientProtocol`:
       - Uses the generated `Client` type from Swift OpenAPI Generator
       - Configured with `URLSessionTransport`
       - Server URL from environment/config (default: staging URL, configurable via Info.plist or build config)
       - Injects `AuthMiddleware` for Bearer token
       - All methods are async throws, using Swift Concurrency
       - Conforms to `@Sendable` requirements for Swift 6 strict concurrency

    4. Create `AuthMiddleware.swift` — `ClientMiddleware` that injects `Authorization: Bearer {token}` header on every request. Reads token from the injected `AuthServiceProtocol`.

    5. Create `EnvelopeUnwrap.swift` — helper to unwrap the NestJS `{ data, meta: { disclaimer } }` response envelope. Provide a generic function: `func unwrapEnvelope<T: Decodable>(_ response: ...) throws -> T` that extracts the `data` field.

    **WWCore — Auth:**

    6. Create `AuthService.swift` — `@Observable` class conforming to `AuthServiceProtocol`:
       - `var token: String?` (tracked by Observation)
       - `var isAuthenticated: Bool { token != nil }`
       - `silentAuth()`: tries keychain first, then calls `POST /v1/auth/anonymous`, saves token to keychain
       - Retry logic: if anonymous auth fails, don't block the user. Set a flag `authPending = true` and retry later
       - Mark as `@MainActor` since it drives UI state

    7. Create `KeychainStore.swift` — concrete `KeychainStoreProtocol` using KeychainAccess library:
       - Service name: "com.wonderwaltz.app"
       - Save/get/delete JWT token
       - Use `.whenUnlockedThisDeviceOnly` accessibility

    **WWDesignSystem:**

    8. Copy `packages/design-tokens/generated/WWDesignTokens.swift` into `apps/ios/Packages/WWDesignSystem/Sources/WWDesignSystem/WWDesignTokens.swift`. Ensure all constants are `public static let`.

    9. Create `Theme.swift` — semantic color helpers:
       - `WWTheme.background` -> `WWDesignTokens.colorSemanticBackground` (or map from primitives)
       - `WWTheme.textPrimary`, `WWTheme.textSecondary`
       - `WWTheme.accent` -> gold
       - `WWTheme.surface` -> cream
       - Dark mode: for v1, light mode only. Provide semantic names so dark mode can be added later.

    10. Create `Typography.swift`:
        - Download Fraunces (variable) and Inter (variable) font files. Add to a Fonts/ resources directory in the package.
        - Register fonts via the package's resource bundle
        - Provide semantic font helpers: `WWTypography.title`, `WWTypography.headline`, `WWTypography.body`, etc.
        - CRITICAL: Use `Font.custom("Fraunces", size: X, relativeTo: .title)` with `relativeTo:` for Dynamic Type scaling. Never use fixed sizes without relativeTo.

    11. Create reusable components in Components/:
        - `WWButton.swift`: Primary (gold bg, navy text) and secondary (outlined) button styles. Min 44pt tap target. Uses `WWTypography` for label font. Supports loading state (spinner replaces label).
        - `WWCard.swift`: Rounded card container with shadow, cream background. Accepts any content via ViewBuilder. Used by wizard steps and plan items.
        - `WWProgressBar.swift`: Horizontal progress bar with gold fill on cream track. Accepts `progress: Double` (0-1) and optional step count.
        - `WWBlurOverlay.swift`: `.ultraThinMaterial` background with centered content slot. Used for locked day overlay. Takes `onAction` closure for CTA tap.

    **WWAnalytics:**

    12. Create `CrashReporting.swift`:
        - `CrashReportingService` class with `static func initialize(dsn: String)` that calls `SentrySDK.start { options in options.dsn = dsn; options.enableSwiftAsyncStacktraces = true; options.tracesSampleRate = 1.0 }`
        - Call this BEFORE the first view renders (in App init or willFinishLaunching equivalent)

    13. Create `AnalyticsService.swift` — concrete `AnalyticsProtocol` wrapping PostHog:
        - `PostHogAnalyticsService` class
        - `initialize(apiKey:host:)` static method
        - `capture(_:properties:)` — CRITICAL: filter out forbidden properties before sending. Forbidden set: `["guest_age", "age_bracket", "guest_age_bracket", "age", "dob", "birthdate", "birth_date"]`. Case-insensitive check.
        - `identify(_:)` — calls PostHogSDK.shared.identify
        - `reset()` — calls PostHogSDK.shared.reset

    **String Catalog:**

    14. Create `Localizable.xcstrings` in the main app target's Resources/. Initialize as an empty String Catalog file. Xcode will auto-discover LocalizedStringKey usage from SwiftUI views in subsequent plans. Use `String(localized:)` in non-view Swift code.

    15. Run `cd apps/ios && tuist generate` then `xcodebuild build -scheme WonderWaltz -destination 'platform=iOS Simulator,name=iPhone 16' -quiet` to verify everything compiles.
