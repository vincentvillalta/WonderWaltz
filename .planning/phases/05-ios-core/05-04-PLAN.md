       - "Back" button (top-left, except on step 0) and "Next"/"Submit" button (bottom)

    2. Create `WizardViewModel.swift` — `@Observable @MainActor`:
       - `var currentStep: WizardStep` — enum with cases: `.dates`, `.parks`, `.guests`, `.budget`, `.lodging`, `.mustDoRides`, `.mealPrefs`, `.review` (8 steps per IOS-06)
       - All wizard data as properties: `startDate`, `endDate`, `selectedParkIds`, `hasHopper`, `guests: [GuestInput]`, `budgetTier`, `lodgingType`, `transportType`, `mustDoRideIds`, `mealPreferences`

       - `var isSubmitting: Bool`
       - `var submissionError: String?`
       - Init: load existing WizardDraft from OfflineStoreProtocol if exists, restore currentStep
       - `func advanceStep()`: save draft to SwiftData (via OfflineStoreProtocol), advance to next step

       - If auth is not available when submitting: show the auth banner per CONTEXT.md ("No internet -- connect to save your trip")
       - Inject `APIClientProtocol` and `OfflineStoreProtocol` via init
