  <action>
  1. Create PlanViewModel.swift: @Observable @MainActor class.
     Properties:
     - plan: the plan data (from API or cached) -- use a PlanDisplayData struct that normalizes API/cached shapes
     - selectedDayIndex: Int (which day tab is active)
     - isLoading: Bool
     - isRethinking: Bool
     - error: String?
     - offlineMode: Bool (true when using cached data)
     
     Methods:
     - loadPlan(planId: String) async: Try API first. On success, cache via SyncCoordinator, trigger OfflinePackageDownloader, and call NotificationPermissionHelper.requestIfNeeded(). On network failure, fall back to SwiftData cached data and set offlineMode = true.
     - rethinkToday(tripId: String, currentTime: String, completedItemIds: [String]) async: POST /trips/:id/rethink-today, set isRethinking = true, poll GET /plans/:id until updated, animate item transitions. If offline, show gentle banner "Connect to internet to rethink your day" (per CONTEXT.md).
     - selectedDay: computed from plan.days[selectedDayIndex]
     - isLocked(dayIndex: Int) -> Bool: returns true for Days 2+ when free tier (entitlement_state != unlocked)
     
     Takes: AuthService (for token), SyncCoordinator (for caching), ModelContext (for offline queries).

  2. Create NotificationPermissionHelper.swift:
     - Static method requestIfNeeded() async that:
       a. Checks UserDefaults key `hasRequestedNotificationPermission` -- if true, return immediately (only ask once)
       b. Calls UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .sound, .badge])
       c. Sets UserDefaults `hasRequestedNotificationPermission = true` regardless of user's choice
     - The system alert is triggered by the requestAuthorization call. Before calling it, present a pre-prompt context view (custom SwiftUI alert or sheet) that says: "Want reminders for Lightning Lane booking windows?" with "Enable Notifications" and "Not Now" buttons. Only call requestAuthorization if user taps "Enable Notifications". If "Not Now", still set the UserDefaults flag so we don't re-ask.
     - Per CONTEXT.md locked decision: this is called after the user sees their Day 1 plan for the first time -- triggered from PlanViewModel.loadPlan() on first successful plan load.

  3. Create PlanContainerView.swift:
     - ForecastBanner at top: "Wait time forecasts are in beta -- times shown are estimates based on historical patterns" (per CONTEXT.md: single info banner, NOT per-item badges)
     - Day tab picker: horizontal ScrollView of day buttons (Mon Jun 15, Tue Jun 16, etc.) with park color accent on selected day
     - Content: DayTimelineView for selected day, or LockedDayOverlay if day is locked
     - "Rethink My Day" floating button at bottom (only shown on unlocked current day)
     - Toolbar: trip name, share button placeholder

  4. Create DayTimelineView.swift:
     - Vertical ScrollView of PlanItemCards
     - Between each item card, show walking time inline: small row with walking icon + "5 min walk" text in muted color
     - Narrative intro text at top of day (from plan day's narrative_intro field)
     - Empty state if no items
     - Pull-to-refresh via .refreshable to reload plan from API

  5. Create PlanItemCard.swift: polymorphic card based on item type.
     - Ride: icon + name + park color stripe + wait time badge + start/end time + LL badge if applicable
     - Meal: fork icon + restaurant name + cuisine tag + time + "Mobile Order" badge if quick-service
     - Show: star icon + show name + time + "Arrive early" note if applicable
     - LL Reminder: bell icon + "Book Lightning Lane" + ride name + time window
     - Rest: coffee icon + "Rest Break" + duration + location suggestion
     - Walk: footprints icon + "Walk to [destination]" + duration (these are the inline walking segments)
     
     Each card uses WWCard component, brand colors, semantic fonts. Tap navigates to detail view (in Task 2).
     44pt minimum tap target. VoiceOver accessibilityLabel with full description.

  6. Create ForecastBanner.swift:
     - Info icon + "Beta Forecast" text + brief explanation
     - Dismissible (hides for the session, re-appears next launch)
     - Cream background with navy text, gold accent border
     - Per CONTEXT.md: one-time framing rather than per-item noise

  7. Write PlanViewTests.swift:
