  <files>

  <action>
  1. Create AuthService.swift: @Observable @MainActor class.
     Properties: token (String?), userId (String?), isAuthenticated (Bool computed), authError (String?).
     Private: keychain (KeychainStore), let apiClient lazy from APIClientProvider.
     
     Methods:
     - silentAuth() async: Try keychain.getToken() first. If cached token exists, set self.token and return. Otherwise call POST /v1/auth/anonymous via the generated API client. On success, save token to keychain, set self.token and self.userId. On failure, set authError but do NOT block — user can browse without auth.
     - logout(): keychain.deleteToken(), token = nil, userId = nil.
     - refreshTokenIfNeeded() async: stub for future token refresh logic.

  2. Update AppState.swift: @Observable @MainActor class.
     Properties:
     - authService: AuthService
     - hasCompletedOnboarding: Bool (persisted via UserDefaults, NOT keychain)
     - navigationState: enum AppScreen { case splash, onboarding, main }
     
     Init creates AuthService. Expose computed isAuthenticated forwarding to authService.

  3. Create SplashView.swift:
     - Shows WonderWaltz logo/brand mark centered on cream background
     - On .task, calls appState.authService.silentAuth()
     - After auth completes (success or failure), transitions:
       - If !hasCompletedOnboarding -> .onboarding
       - Else -> .main
     - Minimum 1.5 second display to prevent flash
     - Use withAnimation for transition

  4. Wire AppState into WonderWaltzApp.swift as @State. Pass to RootView (created in Task 2) via .environment().
