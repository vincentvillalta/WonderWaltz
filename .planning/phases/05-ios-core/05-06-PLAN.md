  <action>
  1. Create/update AccessibilityHelpers.swift:
     - View extension: .withReducedMotion(animation:) that checks @Environment(\.accessibilityReduceMotion) and returns .default or nil
     - View modifier: .adaptiveBlur() that uses .ultraThinMaterial when reduce transparency is off, solid Color with high opacity when on
     - ViewModifier: AccessibleCard that adds proper accessibilityElement grouping for card components
     - Helper: func accessibleTimeDescription(_ time: String) -> String that converts "09:30" to "nine thirty AM" for VoiceOver
     - Helper: func accessibleDurationDescription(_ minutes: Int) -> String that converts 45 to "forty-five minutes"

  2. Audit and fix Typography.swift:
     - Ensure ALL Font.ww* helpers use Font.custom(_:size:relativeTo:) — never bare Font.custom(_:size:)
     - Font.wwLargeTitle: Fraunces 34pt relativeTo .largeTitle
     - Font.wwTitle: Fraunces 28pt relativeTo .title
     - Font.wwTitle2: Fraunces 22pt relativeTo .title2
     - Font.wwHeadline: Inter 17pt relativeTo .headline (bold)
     - Font.wwBody: Inter 17pt relativeTo .body
     - Font.wwCallout: Inter 16pt relativeTo .callout
     - Font.wwCaption: Inter 12pt relativeTo .caption
     - If Fraunces/Inter font files are not bundled yet, use .system with the same sizes (fonts will be bundled when font files are obtained). The relativeTo parameter still works with system fonts as fallback.

  3. Audit all wizard step views:
     - Every Text uses Font.ww* helpers (not hardcoded sizes)
     - Every Button/tappable element has .accessibilityLabel with descriptive text
     - DatePicker fields have .accessibilityLabel("Trip start date") / ("Trip end date")
     - Park selection cards: .accessibilityAddTraits(.isButton) + .accessibilityLabel("Magic Kingdom, selected/not selected")
     - Guest fields: grouped with .accessibilityElement(children: .contain)
     - Budget tier cards: .accessibilityLabel("Pixie Dust tier: value-focused, no Lightning Lane") etc.
     - Review step sections: each summary card is an .accessibilityElement with full summary as label
     - No text truncation at accessibility5 — use ScrollView if content overflows, .lineLimit(nil) on multi-line text
     - Wizard progress bar: .accessibilityLabel("Step \(n) of 8") .accessibilityValue("\(Int(progress * 100)) percent complete")

  4. Audit onboarding views:
     - Page indicator: .accessibilityLabel("Page \(n) of 3")
     - Skip button: .accessibilityLabel("Skip onboarding")
     - Page transitions: respect reduce motion (no animation if enabled)
     - All text: Font.ww* helpers, no truncation at accessibility5
