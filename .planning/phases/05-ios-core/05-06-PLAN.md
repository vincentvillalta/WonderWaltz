  <name>Task 1: Accessibility utility modifiers + design system audit</name>
  <files>
    apps/ios/Packages/WWDesignSystem/Sources/WWDesignSystem/Modifiers/AccessibilityModifiers.swift

    apps/ios/Packages/WWDesignSystem/Sources/WWDesignSystem/Components/WWBlurOverlay.swift
    apps/ios/Packages/WWDesignSystem/Sources/WWDesignSystem/Typography.swift
  </files>
  <action>
    1. Create `AccessibilityModifiers.swift` — reusable view modifiers:
       - `wwAccessibleTapTarget()`: ensures minimum 44x44pt hit area via `.frame(minWidth: 44, minHeight: 44)` and `.contentShape(Rectangle())`
       - `wwAccessibilityAction(label:action:)`: wraps common accessibility action pattern
       - Extension on `View` for `.wwAccessibilityLabel(_ text: LocalizedStringKey)` that applies both `.accessibilityLabel` and `.accessibilityAddTraits(.isButton)` for interactive elements
       - `wwAccessibleHidden()`: shorthand for `.accessibilityHidden(true)` on decorative elements

    2. Create `ReduceMotionModifier.swift`:
       - `conditionalAnimation<V: Equatable>(value: V)` modifier that checks `@Environment(\.accessibilityReduceMotion)`:
         - If reduce motion ON: `.animation(.none, value: value)` — instant transition
         - If reduce motion OFF: `.animation(.spring, value: value)` — normal animation
       - `wwTransition(_ transition: AnyTransition)` modifier: applies transition only when reduce motion is OFF

    3. Audit WWDesignSystem components:
       - `WWButton.swift`: verify min 44pt tap target via frame. Add `.accessibilityAddTraits(.isButton)`. Ensure loading state announces "Loading" to VoiceOver
       - `WWCard.swift`: verify it doesn't trap focus (interactive cards need `.accessibilityElement(children: .combine)` to group label)
       - `WWProgressBar.swift`: add `.accessibilityValue("Step \(current) of \(total)")`. Add `.accessibilityAddTraits(.updatesFrequently)` if it animates
       - `WWBlurOverlay.swift`: add `.accessibilityAddTraits(.isModal)` so VoiceOver focuses within the overlay. Check `.accessibilityElement(children: .contain)` for overlay content
       - Typography.swift: verify ALL font definitions use `relativeTo:` parameter. No fixed `Font.system(size:)` calls

    4. Color contrast audit:
       - Gold on Cream: per Phase 1 decision, gold is decorative/background only, NEVER body text (fails WCAG at 2.1:1)
       - Navy on Cream: verify 4.5:1+ ratio for body text
       - White on Gold: verify for button labels
       - All semantic text colors in Theme.swift must meet WCAG 2.2 AA (4.5:1 normal text, 3:1 large text)
       - If any color pair fails: adjust in Theme.swift with a comment noting the contrast ratio

    **Intermediate build verification:** After completing the design system audit, run `xcodebuild build` to verify all changes compile before proceeding to Task 2.

  <action>
    Systematic audit of every screen. For each view, check and fix:

    **A. VoiceOver labels (all screens):**
    - Every interactive element (button, toggle, picker, card) has `.accessibilityLabel()` with descriptive text
    - Decorative images have `.accessibilityHidden(true)`
    - Grouped content uses `.accessibilityElement(children: .combine)` where appropriate
    - Navigation is linear — VoiceOver doesn't get stuck in loops or skip content
    - Custom controls (park selection cards, budget tier cards) have `.accessibilityAddTraits(.isButton)` and announce selection state

    **B. Dynamic Type at accessibility5 (all screens):**
    - Set simulator to Settings > Accessibility > Larger Text > max size
    - Verify no text clips or truncates. Use `.lineLimit(nil)` where text might overflow
    - ScrollView wraps content that might exceed screen height at large sizes
    - Layout adjusts: horizontal stacks that clip should switch to VStack at large Dynamic Type via `@Environment(\.dynamicTypeSize)`
    - Example: park selection grid (2 columns) should become single column at accessibility5
    - Review step summary should remain scrollable at any text size

    **C. Reduce motion (animated screens):**
    - Apply `conditionalAnimation` modifier from Task 1 to:
      - Wizard step transitions
      - Rethink loading overlay appearance/dismissal (RethinkLoadingView)
      - Plan item position changes after rethink
      - Onboarding page transitions
      - Progress bar fill animation
    - When reduce motion ON: all transitions are instant (no spring/ease)

    **D. Reduce transparency:**
    - Check `@Environment(\.accessibilityReduceTransparency)` in:
      - LockedDayOverlay: if reduce transparency ON, use solid background instead of `.ultraThinMaterial`
      - Any views using `.opacity()` for visual hierarchy: ensure content remains readable

    **E. Minimum tap targets (44pt):**
    - Apply `.wwAccessibleTapTarget()` to all interactive elements that might be undersized:
      - Day tab picker tabs
      - Park selection cards
      - Guest add/remove buttons
      - Ride picker search results
      - Plan item cards
      - Rethink button
      - Onboarding page dots (if tappable)

    **F. Focus management:**
    - After wizard step transition: move VoiceOver focus to the new step's title
    - After rethink completes: announce "Plan updated" via `UIAccessibility.post(notification: .announcement, argument: "Plan updated")`
    - LockedDayOverlay: trap focus within overlay when displayed (`.accessibilityAddTraits(.isModal)`)
