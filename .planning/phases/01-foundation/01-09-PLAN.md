```markdown
# Accessibility — WonderWaltz

## Standard: WCAG 2.2 AA

All user-facing surfaces (iOS, Android, Web) must meet WCAG 2.2 Level AA.

## Color Contrast

| Context | Minimum Ratio |
|---|---|
| Normal text (< 18pt or < 14pt bold) | 4.5:1 |
| Large text (≥ 18pt or ≥ 14pt bold) | 3:1 |
| UI components, icons, focus indicators | 3:1 |
| Disabled states | No minimum (but must look disabled) |

**Verification:** Use Stark (Figma plugin) or WCAG contrast checker on all color pairs.
Never rely on color alone to convey information — add icons, labels, or patterns.

## Tap Targets (iOS / Android)

- Minimum: 44×44pt (iOS HIG) / 48×48dp (Material Design)
- Recommended: 48×48pt (matches both HIG and Material)
- All interactive elements must meet this minimum
- Use padding, not resizing the visual element, to meet the minimum

## Focus Indicators (Web)

- Custom focus rings must have at least 3:1 contrast against adjacent colors
- Focus must not be obscured by sticky headers or floating elements (WCAG 2.2 SC 2.4.11, 2.4.12)
- Tab order must follow reading order

## Dynamic Type

- **iOS:** Support `accessibility5` (largest accessibility text size) without clipping
  - Use `scaledFont(for:)` in SwiftUI; test at `accessibility5` setting
  - No fixed heights on text containers — use `minHeight` with flexibility
- **Android:** Support largest Accessibility font scale
  - Use `sp` units for all text sizes (scales with system font size)

## VoiceOver (iOS) and TalkBack (Android)

- Every interactive element must have an accessibility label
- Decorative images: `accessibilityHidden = true` / `contentDescription = null`
- Custom controls must declare accessibilityRole
- Grouped items should be combined (reduce VoiceOver swipe count)
- Test every screen with VoiceOver/TalkBack from first implementation

## Reduce Motion (iOS)

Check `UIAccessibility.isReduceMotionEnabled` before running animations.
WonderWaltz uses SwiftUI's `withAnimation` — test all animated transitions with reduce motion on.

## Dragging Alternatives (WCAG 2.2)

Any interaction that requires dragging must have a tap/button alternative:
- Scrollable lists: additional "scroll to top" button if list is very long
- Reorderable items: handle tap to focus → move up/down buttons
- Range sliders: manual text input fallback

---
*Standard: WCAG 2.2 Level AA*
*Applies to: iOS (Phase 5), Android (Phase 7), Web (Phase 8)*
```

```markdown
# Iconography — WonderWaltz

## Icon Library: Phosphor Icons

**License:** MIT — permissive, commercial use allowed
**Packages:**
- Web: `@phosphor-icons/react` (React components)
- iOS: `phosphor-swift` (Swift Package)
- Android: `phosphor-android` (Compose — or use SVG export)

**Why Phosphor:**
- Single library with official iOS, Android, and web packages
- MIT licensed — no attribution required
- 1400+ icons covering all WonderWaltz needs
- Five weights (Thin, Light, Regular, Bold, Fill) for design flexibility
- Phosphor vs Lucide: Both are excellent. Phosphor chosen for broader platform coverage.

## Icon Weight Usage

| Context | Weight | Example |
|---|---|---|
| Navigation bar | Regular | Home, Search, Profile |
| Action buttons | Bold or Fill | Add, Delete, Share |
| Informational | Light or Regular | Info badge, Help |
| Status indicators | Fill | Success ✓, Error ✗, Warning ⚠ |

## Minimum Tap Targets

All icons used as interactive elements must meet:
- iOS: 44×44pt minimum touch target (icon may be 24×24pt with padding)
- Android: 48×48dp minimum touch target
- Web: 44×44px (following iOS HIG minimum as the cross-platform floor)

## WDW-Specific Icons

These icons are commonly needed for WonderWaltz:
- Ride / Attraction: `Roller Coaster` (or `Lightning` for thrill)
- Dining: `Fork Knife` or `Hamburger`
- Show / Entertainment: `Theater Masks`
- Walking / Navigation: `Footprints` or `Map Pin`
- Lightning Lane / Queue: `Lightning` or `Ticket`
- Rest / Break: `Hammock` or `Couch`
- Hotel / Resort: `Buildings` or `Bed`
- Camera / Memory: `Camera`
- Calendar / Date: `Calendar`
- Clock / Time: `Clock`

## Custom Illustration Policy (LEGL-03)

**No Disney trademarks in any graphic asset.** This includes:
- No Mickey Mouse silhouettes (classic ears shape)
- No castle silhouettes or imagery
- No ride photography
- No character art
- No official park maps

Acceptable:
- Original illustrated characters (non-Disney family, generic)
- Abstract patterns suggesting magic, stars, confetti
- Geometric shapes and color blocks
- Phosphor icons as the primary visual vocabulary
- Stock photography of Orlando (non-park-specific)

---
*Icon library: Phosphor Icons (MIT)*
*Applies to: iOS (Phase 5), Android (Phase 7), Web (Phase 8)*
```
